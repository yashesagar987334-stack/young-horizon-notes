# 项目上下文

## 项目定位

这是一个以生活记录与内容整理为核心的个人网站，不强调强烈的个人展示，更强调：

- 生活记录
- 内容分类整理
- 高质量阅读体验
- 后续可持续维护

网站整体基调：

- 简洁
- 克制
- 清爽
- 偏高质量内容排版
- 手机端和电脑端都可访问

当前项目采用静态站思路实现，适合后续通过新增内容文件来维护。

---

## 当前技术方案

- 前端框架：Astro
- 语言：TypeScript
- 内容格式：MDX
- 内容管理：Astro Content Collections
- 样式：SCSS + CSS Variables
- 部署目标：GitHub Pages
- 自动部署：GitHub Actions

当前内容与配置拆分方式：

- `src/content/notes`：文章内容
- `src/data/categories.ts`：分类配置
- `src/data/links.ts`：外部链接配置
- `src/data/puzzles.ts`：拼图游戏图片配置
- `public/images`：静态图片资源

## 当前电子产品均价工具定位

它是站内工具，不是外部独立网站。

当前目标是：

- 放在 `娱乐小站 / 电子产品均价`
- 可以创建多个分析卡片
- 每张卡片单独填写设备信息
- 每张卡片在自身内部显示结果

每张卡片当前字段：

- 产品名称
- 产品类别
- 购买价格
- 购买时间

每张卡片结果：

- 持有时间
- 日均价格

当前不包含：

- 自定义类别
- 残值
- 购买渠道
- 备注
- 图表
- 后端同步

当前新的事实：

- 该工具页已经在站内接通
- 用户当前反馈：手机端和电脑端都能看到页面，但无法正常实际操作
- 因此下一阶段不是继续扩字段，而是优先修复交互失效

## 当前拼图游戏工具定位

它也是站内工具，不是外部独立网站。

当前目标是：

- 放在 `娱乐小站 / 拼图游戏`
- 第一版只保留两张真实图片做测试
- 页面结构为“两个入口卡片 + 一个九宫格拼图盘”
- 点击哪一张图，就在同一个拼图盘里玩哪一张

当前两张测试图为：

- `嘉庚像`
- `芙蓉湖畔`

当前实现方向：

- 固定 `3 x 3` 九宫格
- 同页切换图片
- 提供 `重新打乱`
- 不做难度切换
- 不做计时或步数系统

当前新的事实：

- 拼图页已接入站点并可访问
- 用户当前反馈：手机端和电脑端都能看到页面，但无法正常实际操作
- 因此下一阶段应优先修复交互逻辑，而不是继续扩图片数量

## 当前内容维护方式

站点内容继续遵守“新增文件即可发布”的思路。

当前已经验证过的内容入口：

- `src/content/notes/*.mdx`：新增一篇文章
- `src/data/links.ts`：调整外部链接
- `src/data/categories.ts`：调整分类
- `src/data/puzzles.ts`：维护拼图图片配置

当前仓库里已经加入一篇用于练习重新发布流程的示例文章：

- `src/content/notes/2026-04-learning-first-publish.mdx`

## 维护原则

后续继续推进时，优先遵守以下原则：

- 不随意扩大需求
- 不擅自改变分类体系
- 不新增重型技术栈
- 优先保证可维护性
- 优先把已确定的功能做稳，再继续扩展
- 工具页如果“能显示但不能操作”，优先修复交互，再谈扩展功能
