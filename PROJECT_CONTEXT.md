# 项目上下文

## 项目定位

这是一个以生活记录与内容整理为核心的个人网站，不强调强烈的个人展示，更强调：

- 生活记录
- 内容分类整理
- 高质量阅读体验
- 后续可持续维护

网站整体基调：

- 简洁
- 克制
- 清爽
- 偏高质量内容排版
- 手机端和电脑端都可访问

当前项目采用静态站思路实现，适合后续通过新增内容文件来维护。

---

## 当前技术方案

- 前端框架：Astro
- 语言：TypeScript
- 内容格式：MDX
- 内容管理：Astro Content Collections
- 样式：SCSS + CSS Variables
- 部署目标：GitHub Pages
- 自动部署：GitHub Actions

当前内容与配置拆分方式：

- `src/content/notes`：文章内容
- `src/data/categories.ts`：分类配置
- `src/data/links.ts`：外部链接配置
- `public/images`：静态图片资源

## 当前分类结构

### 1. 我与厦大

描述：
关于厦大里的项目、校园生活，以及毕业前想认真完成的一些事情。

子分类：

- `My Compass`
- `毕业前的 50 件事`

### 2. 日常生活

描述：
关于普通日子、出门玩耍和那些值得被轻轻记下的生活片段。

子分类：

- `日常记录`
- `出去玩`

### 3. 娱乐小站

描述：
放一些轻量小工具、小实验，以及你想单独收纳的实用网页内容。

子分类：

- `电子产品均价`
- `其他`

说明：
`电子产品均价` 当前是工具型分类，不是普通文章分类。

## 当前已经确定的页面方向

- 首页：作为内容总入口
- 分类页：按一级分类和二级分类进入内容
- 记录页：查看全部文章记录
- 详情页：偏公众号阅读风格
- 链接页：放外部链接
- 工具页：当前挂在 `娱乐小站 / 电子产品均价`

## 当前电子产品均价工具的定位

它是站内工具，不是外部独立网站。

当前目标是：

- 放在 `娱乐小站 / 电子产品均价`
- 可以创建多个分析卡片
- 每张卡片单独填写设备信息
- 每张卡片在自身内部显示结果

每张卡片当前字段：

- 产品名称
- 产品类别
- 购买价格
- 购买时间

每张卡片结果：

- 持有时间
- 日均价格

当前不包含：

- 自定义类别
- 残值
- 购买渠道
- 备注
- 图表
- 后端同步

## 维护原则

后续继续推进时，优先遵守以下原则：

- 不随意扩大需求
- 不擅自改变分类体系
- 不新增重型技术栈
- 优先保证可维护性
- 优先把已确定的功能做稳，再继续扩展
